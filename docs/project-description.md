# Конференции.РФ: описание проекта

Информационная система для бронирования помещений под всероссийские конференции. Проект соответствует варианту В2: регистрация, авторизация, личный кабинет, оформление заявок, отзывы и панель администратора.

## Архитектура

```text
project/
├── index.php          # главная страница
├── register.php       # регистрация пользователя
├── login.php          # вход пользователя и администратора
├── logout.php         # выход
├── profile.php        # личный кабинет, история заявок, отзывы
├── booking.php        # оформление заявки
├── admin.php          # панель администратора
├── config.php         # подключение к MySQL и общие функции
├── src/
│   ├── Database.php   # ООП-подключение к MySQL через PDO
│   ├── Auth.php       # авторизация и управление сессией
│   └── RegistrationValidator.php # серверная валидация
├── css/
│   └── vendor/        # локальная библиотека Bootstrap
├── js/
├── images/
├── docs/
└── sql/
    └── init.sql       # создание базы данных для phpMyAdmin
```

## Страницы

- `/index.php` - главная страница портала.
- `/register.php` - регистрация с обязательными полями: логин, пароль, ФИО, телефон, e-mail.
- `/login.php` - авторизация по логину и паролю.
- `/booking.php` - выбор помещения, даты начала конференции и способа оплаты.
- `/profile.php` - история заявок, слайдер из четырех изображений и отзывы.
- `/admin.php` - просмотр всех заявок, фильтр, сортировка, пагинация и смена статуса.

## Роли

Обычный пользователь регистрируется самостоятельно. После входа он может создавать заявки и видеть только свою историю.

Администратор входит с фиксированными данными:

```text
Логин: Admin26
Пароль: Demo20
```

## Backend и база данных

Backend написан на PHP с PDO. Данные хранятся в MySQL:

- `users` - пользователи;
- `rooms` - помещения;
- `bookings` - заявки на бронирование;
- `reviews` - отзывы.

SQL-запросы выполняются через prepared statements. Пароли пользователей хранятся через `password_hash`.

Подключение к БД, аутентификация и проверка регистрационных данных реализованы отдельными классами. Уникальность логина и e-mail проверяется приложением и ограничениями базы данных.

## Дизайн

Интерфейс адаптирован под мобильный экран 390×844 пикселей. Используются готовые классы UI Kit:

- `.btn`, `.btn--ghost`, `.btn--accent`, `.btn--danger`;
- `.card`;
- `.input`, `.select`, `.textarea`, `.field`;
- `.badge`;
- `.grid-3`, `.split`, `.form-grid`;
- `.toast`, `.slider`.

В качестве библиотеки стилей подключена локальная сборка Bootstrap 5.3.8. Авторские стили загружаются после нее и формируют визуальную систему проекта.

Слайдер автоматически переключается каждые 3 секунды и имеет кнопки вперед/назад.
